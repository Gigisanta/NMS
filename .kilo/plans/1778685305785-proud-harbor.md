# Fix: Group Selector Dropdown Background Issue

## Problem

When opening the group selector dropdown in the clients view, the white background only covers part of the list (cuts off halfway). The remaining items appear translucent, overlapping with the UI behind the dropdown.

## Root Cause

In `src/components/modules/group-selector.tsx`:

1. `PopoverContent` has `bg-background` (white) but `p-4` padding, so the background only covers the padded area
2. `ScrollArea` (line 204) has `max-h-[400px]` but its viewport has no explicit background color
3. Items inside `ScrollArea` don't have solid backgrounds, so they show through as translucent

## Fix

**File: `src/components/ui/scroll-area.tsx`**

Add `bg-popover` to the `ScrollAreaPrimitive.Viewport` to ensure the background is solid white and fully opaque:

```tsx
<ScrollAreaPrimitive.Viewport
  data-slot="scroll-area-viewport"
  className="focus-visible:ring-ring/50 size-full rounded-[inherit] transition-[color,box-shadow] outline-none focus-visible:ring-[3px] focus-visible:outline-1 bg-popover"
>
  {children}
</ScrollAreaPrimitive.Viewport>
```

**File: `src/components/ui/popover.tsx`**

Ensure `PopoverContent` has `overflow-hidden` to prevent content from bleeding outside the rounded corners:

```tsx
className={cn(
  "bg-popover text-popover-foreground data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 data-[state=closed]:zoom-out-95 data-[state=open]:zoom-in-95 data-[side=bottom]:slide-in-from-top-2 data-[side=left]:slide-in-from-right-2 data-[side=right]:slide-in-from-left-2 data-[side=top]:slide-in-from-bottom-2 z-50 w-72 origin-(--radix-popover-content-transform-origin) rounded-md border p-4 shadow-md outline-hidden overflow-hidden",
  className
)}
```

## Files to Modify

1. `src/components/ui/scroll-area.tsx` - Add `bg-popover` to viewport
2. `src/components/ui/popover.tsx` - Add `overflow-hidden` to content

## Verification

- No translucent/overlapping items
