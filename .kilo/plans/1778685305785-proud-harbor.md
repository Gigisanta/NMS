# Fix: Group Selector Dropdown Scroll Issue

## Problem

The dropdown's scrollbar is visually confused with the page scrollbar, and when scrolling inside the dropdown, the page scroll activates instead of the dropdown's own scroll.

## Root Cause

Radix `ScrollArea` uses custom scrollbar rendering, but the wheel event handling can propagate to the parent page scroll when:
1. The `ScrollArea` viewport doesn't have `overscroll-behavior: contain` to stop event propagation
2. The viewport might not be properly capturing wheel events
3. The custom scrollbar thumb (`bg-slate-300`) may not have explicit dimensions, making it invisible

## Fix

**File: `src/components/ui/scroll-area.tsx`**

Update the `ScrollAreaPrimitive.Viewport` to add `overscroll-behavior-contain` and ensure the viewport properly contains scroll:

```tsx
<ScrollAreaPrimitive.Viewport
  data-slot="scroll-area-viewport"
  className="focus-visible:ring-ring/50 size-full rounded-[inherit] transition-[color,box-shadow] outline-none focus-visible:ring-[3px] focus-visible:outline-1 bg-popover overscroll-contain"
>
  {children}
</ScrollAreaPrimitive.Viewport>
```

Also update the `ScrollBar` thumb to have explicit dimensions and ensure it's always visible:

```tsx
<ScrollAreaPrimitive.ScrollAreaThumb
  data-slot="scroll-area-thumb"
  className="bg-slate-300 hover:bg-slate-400 relative flex-1 rounded-full min-w-full min-h-[60px]"
/>
```

**File: `src/components/ui/popover.tsx`**

The popover already has `overflow-hidden`, but we should ensure `max-h` is set properly. However, since `max-h` is set on the group-selector's `PopoverContent` (`max-h-[70vh]`), this should work.

## Files to Modify

1. `src/components/ui/scroll-area.tsx` - Add `overscroll-contain` to viewport and explicit min dimensions to thumb

## Verification

After fix:
- Dropdown has its own independent scrollbar on the right side
- Scrolling inside dropdown does NOT scroll the page
- Scrollbar is visually distinct and clearly belongs to the dropdown
