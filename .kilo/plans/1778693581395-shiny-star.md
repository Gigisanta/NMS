# Plan: Add visible Celeste (light blue) scrollbar to GroupSelector dropdown

## Problem
The `GroupSelector` dropdown cannot scroll to show all groups because:
1. `overflow-hidden` on `PopoverContent` clips the ScrollArea's scrollbar
2. The scrollbar color doesn't contrast well with the background

## Root Cause
In `src/components/modules/group-selector.tsx` line 134:
```tsx
className="w-72 p-0 z-[9999] bg-background border-shadow-md max-h-[70vh] overflow-hidden flex flex-col"
```
The `overflow-hidden` clips the Radix UI ScrollArea's scrollbar (which renders outside the viewport).

## Solution
Two changes needed:

### 1. Change `overflow-hidden` to `overflow-clip` in group-selector.tsx (line 134)
```tsx
// Antes:
className="... overflow-hidden flex flex-col"
// Después:
className="... overflow-clip flex flex-col"
```

### 2. Make scrollbar Celeste (light blue) in scroll-area.tsx (line 43 and 52)
```tsx
// Line 43 - border color:
orientation === "vertical" && "h-full w-2.5 border-l border-l-slate-300"

// Line 52 - thumb color:
className="bg-sky-300 hover:bg-sky-400 relative flex-1 rounded-full min-w-full min-h-[60px]"
```

## Files to modify:
1. `src/components/modules/group-selector.tsx` - line 134: `overflow-hidden` → `overflow-clip`
2. `src/components/ui/scroll-area.tsx` - lines 43, 52: Change to Celeste (#87CEEB or similar sky color)

## Verification
After changes, the dropdown should have a visible light blue scrollbar on the right side that allows scrolling through all groups.
